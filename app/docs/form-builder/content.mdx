# SchemaForm

A schema-driven form builder. Define your form as a config array — sections, fields, layout, and buttons — and `SchemaForm` handles rendering, validation, loading states, and submission.

## Install

<InstallCmd file="form-dynamic-template.json" />

<Callout variant="info">
  This block includes `SchemaForm`, `FieldRenderer`, and all supporting utilities. Install this if you want the full form builder experience.
</Callout>

<Callout variant="tip">
  The registry install handles everything automatically — npm packages (`react-hook-form`, `@hookform/resolvers`, `zod`, `sonner`, `react-google-recaptcha`) and shadcn components (`field`, `button`, `card`, `sonner`). No manual installs needed.
</Callout>

After installing, add the `Toaster` once in your root layout. Make sure `font-sans` is on `<body>` or your app will render with a serif system font:

```tsx
// app/layout.tsx
import { Geist, Geist_Mono } from "next/font/google"
import { Toaster } from "@/components/ui/sonner"
import "./globals.css"

const geistSans = Geist({ variable: "--font-geist-sans", subsets: ["latin"] })
const geistMono = Geist_Mono({ variable: "--font-geist-mono", subsets: ["latin"] })

export default function RootLayout({ children }: { children: React.ReactNode }) {
  return (
    <html lang="en">
      <body className={`${geistSans.variable} ${geistMono.variable} font-sans antialiased`}>
        {children}
        <Toaster position="top-center" />
      </body>
    </html>
  )
}
```

## Usage

<Callout variant="warning">
  `SchemaForm` uses React hooks internally — add `"use client"` to any file that renders it.
</Callout>

```tsx
"use client"

import { SchemaForm } from "@/components/forms/form-ui/standard-form"
import { FormFieldType } from "@/components/forms/core"
import { z } from "zod"

const schema = z.object({
  firstName: z.string().min(1, "Required"),
  lastName: z.string().min(1, "Required"),
  email: z.string().email(),
})

export function ProfileForm() {
  return (
    <SchemaForm
      schema={schema}
      defaultValues={{ firstName: "", lastName: "", email: "" }}
      sections={[
        {
          title: "Personal Info",
          layout: "grid-2",
          fields: [
            { name: "firstName", fieldType: FormFieldType.INPUT, label: "First name" },
            { name: "lastName", fieldType: FormFieldType.INPUT, label: "Last name" },
            { name: "email", fieldType: FormFieldType.INPUT, label: "Email", required: true, colSpan: "full" },
          ],
        },
      ]}
      buttons={{
        submit: {
          label: "Save profile",
          submit: true,
          loadingLabel: "Saving…",
          onSubmit: async (values, form) => {
            await saveProfile(values)
          },
        },
      }}
    />
  )
}
```

## Section config

Each item in `sections` configures a visual grouping of fields:

```ts
interface SchemaSectionConfig<T> {
  title?: string           // Section heading
  description?: string     // Section sub-heading
  layout?: SectionLayout   // Controls the field grid
  columns?: number         // Column count when layout is "grid" (1–6)
  fields: SchemaFieldConfig<T>[]
}

type SectionLayout =
  | "stack"     // Single column (default)
  | "grid"      // Dynamic grid — use columns prop to set count
  | "grid-2"    // 2-column grid
  | "grid-3"    // 3-column grid
  | "grid-4"    // 4-column grid
  | "grid-6"    // 6-column grid
  | "auto-fit"  // Auto-fit columns based on container width
```

## Button config

```ts
interface FormActionConfig<T> {
  label: string                  // Button text
  submit?: boolean               // true = triggers form submit
  loadingLabel?: string          // Text shown while onSubmit is pending
  variant?: ButtonVariant        // shadcn button variant
  iconLeft?: ReactNode           // Icon rendered left of label
  iconRight?: ReactNode          // Icon rendered right of label
  className?: string             // Additional button classes
  onSubmit?: (values: T, form: UseFormReturn<T>) => void | Promise<void>
}
```

Multiple buttons are supported — each can have its own `onSubmit` handler:

```tsx
buttons={{
  save: {
    label: "Save draft",
    submit: true,
    loadingLabel: "Saving…",
    onSubmit: async (values, form) => saveDraft(values),
  },
  publish: {
    label: "Publish",
    submit: true,
    loadingLabel: "Publishing…",
    variant: "default",
    onSubmit: async (values, form) => publish(values),
  },
}}
```

## CRUD modes

```tsx
// CREATE — blank form, resets after successful submit
<SchemaForm formType="CREATE" ... />

// UPDATE — pre-filled, all fields editable
<SchemaForm formType="UPDATE" defaultValues={existing} ... />

// VIEW — all fields rendered as read-only
<SchemaForm formType="VIEW" defaultValues={existing} ... />
```

<Callout variant="tip">
  In `VIEW` mode all fields are automatically disabled — no extra config needed per field.
</Callout>

## Conditional fields

Use the `show` prop on any field to conditionally render it based on current form values:

```tsx
{
  name: "companyName",
  fieldType: FormFieldType.INPUT,
  label: "Company name",
  show: (values) => values.accountType === "business",
}
```

## reCAPTCHA

Set `recaptcha={true}` to show the reCAPTCHA widget above the submit buttons. Submission is blocked until the user completes verification.

```tsx
<SchemaForm
  recaptcha
  schema={schema}
  defaultValues={defaults}
  sections={sections}
  buttons={buttons}
/>
```

Add your site key to `.env.local`:

```bash
NEXT_PUBLIC_RECAPTCHA_SITE_KEY=your_site_key_here
```
