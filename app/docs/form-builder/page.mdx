# FormBuilderStandard

A schema-driven form builder. Define your form as a config array — sections, fields, layout, and buttons — and `FormBuilderStandard` handles rendering, validation, loading states, and submission.

## Install

<InstallCmd file="form-dynamic-template.json" />

</Callout>

## Usage

```tsx
import { FormBuilderStandard } from "@/components/forms/form-ui/standard-form/FormBuilderStandard"
import { FormFieldType } from "@/components/forms/form-field/constants"
import { z } from "zod"

const schema = z.object({
  firstName: z.string().min(1, "Required"),
  lastName: z.string().min(1, "Required"),
  email: z.string().email(),
})

export function ProfileForm() {
  return (
    <FormBuilderStandard
      schema={schema}
      defaultValues={{ firstName: "", lastName: "", email: "" }}
      sections={[
        {
          title: "Personal Info",
          layout: "grid-2",
          fields: [
            { name: "firstName", fieldType: FormFieldType.INPUT, label: "First name" },
            { name: "lastName", fieldType: FormFieldType.INPUT, label: "Last name" },
            { name: "email", fieldType: FormFieldType.INPUT, label: "Email", required: true },
          ],
        },
      ]}
      buttons={{
        submit: {
          label: "Save profile",
          submit: true,
          loadingLabel: "Saving…",
          onSubmit: async (values) => {
            await saveProfile(values)
          },
        },
      }}
    />
  )
}
```

## Section config

Each item in `sections` configures a visual grouping of fields:

```ts
interface FormBuilderSection<T> {
  title?: string           // Section heading
  description?: string     // Section sub-heading
  layout?: SectionLayout   // Controls the field grid
  fields: FormBuilderField<T>[]
}

type SectionLayout =
  | "stack"     // Single column (default)
  | "grid"      // 2-column grid
  | "grid-2"    // 2-column grid
  | "grid-3"    // 3-column grid
  | "grid-4"    // 4-column grid
  | "grid-6"    // 6-column grid
  | "auto-fit"  // Auto-fit columns based on container width
```

## Button config

```ts
interface FormBuilderButton<T> {
  label: string               // Button text
  submit?: boolean            // true = triggers form submit + onSubmit
  loadingLabel?: string       // Text shown while onSubmit is pending
  variant?: ButtonVariant     // shadcn button variant
  onSubmit?: (values: T) => void | Promise<void>
}
```

Multiple buttons are supported — each can have its own `onSubmit` handler:

```tsx
buttons={{
  save: {
    label: "Save draft",
    submit: true,
    loadingLabel: "Saving…",
    onSubmit: async (values) => saveDraft(values),
  },
  publish: {
    label: "Publish",
    submit: true,
    loadingLabel: "Publishing…",
    onSubmit: async (values) => publish(values),
  },
}}
```

## CRUD modes

```tsx
// CREATE — blank form, resets after successful submit
<FormBuilderStandard formType="CREATE" ... />

// UPDATE — pre-filled, all fields editable
<FormBuilderStandard formType="UPDATE" defaultValues={existing} ... />

// VIEW — all fields rendered as read-only
<FormBuilderStandard formType="VIEW" defaultValues={existing} ... />
```

<Callout variant="tip">
  In `VIEW` mode, all `DynamicFormField` instances receive `disabled={true}` automatically — no extra config needed.
</Callout>
