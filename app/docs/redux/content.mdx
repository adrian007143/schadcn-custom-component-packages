# Redux Methods Tool

The Redux package installs a lightweight persisted store scaffold built around `react-redux-methods`.

## Install

<InstallCmd file="redux-methods-tool.json" />

## What Gets Installed

```bash
redux/
├── store.ts
├── StoreProvider.tsx
├── connections.ts
└── reducers/
    ├── index.ts
    ├── notification.ts
    ├── sidebar.ts
    ├── todo.ts
    └── theme.ts

lib/
├── helper/storageLocal.ts
└── theme/types.ts
```

## Redux-Only Root Setup

If Redux is the only package you installed, wrap the app with `StoreProvider`.

```tsx
// app/layout.tsx
import StoreProvider from "@/redux/StoreProvider"

export default function RootLayout({
  children,
}: {
  children: React.ReactNode
}) {
  return (
    <html lang="en">
      <body>
        <StoreProvider>{children}</StoreProvider>
      </body>
    </html>
  )
}
```

## Redux Plus Theme Builder

If your app also installs `theme-preset-tool`, keep the providers explicit:

```tsx
import StoreProvider from "@/redux/StoreProvider"
import { AppProviders } from "@/components/providers/AppProviders"
import { AppHeader } from "@/components/site/AppHeader"
import { AppShell } from "@/components/site/AppShell"

export default function RootLayout({
  children,
}: {
  children: React.ReactNode
}) {
  return (
    <html lang="en" suppressHydrationWarning>
      <body suppressHydrationWarning>
        <StoreProvider>
          <AppProviders>
            <AppShell header={<AppHeader />}>{children}</AppShell>
          </AppProviders>
        </StoreProvider>
      </body>
    </html>
  )
}
```

<Callout variant="info">
  `AppProviders` belongs to the theme package and manages theme UI only. `StoreProvider` remains the Redux root for state slices installed by `redux-methods-tool`.
</Callout>

## Reading State

Use `useGetState` inside client components:

```tsx
"use client"

import { useGetState } from "@/redux/connections"

export function TodoCount() {
  const count = useGetState.getTodoCount()

  return <span>{count}</span>
}
```

## Dispatching Actions

Use `dispatch` for action creators exported from the generated reducers:

```tsx
"use client"

import { dispatch } from "@/redux/connections"

export function AddTodoButton() {
  return (
    <button onClick={() => dispatch.addTodo("Ship docs")}>
      Add Todo
    </button>
  )
}
```

## Persistence

The generated store persists selected slices through `persistKeys` in `redux/store.ts`.

Default persisted slices:

- `notifications`
- `sidebar`
- `todo`
- `theme`

Update `persistKeys` if your app should persist fewer or additional slices.
