# Redux Setup

A complete Redux configuration using `react-redux-methods` — a zero-boilerplate wrapper around Redux that auto-generates actions, selectors, and typed hooks from a single `createReduxMethods` call. Includes localStorage persistence and Redux DevTools support.

## Install

<InstallCmd file="redux-methods-tool.json" />

## What's installed

```
redux/
├── store.ts              Redux store with DevTools + localStorage persistence
├── StoreProvider.tsx     React context wrapper — wrap your app root with this
├── connections.ts        Typed dispatch / selector hooks
└── reducers/
    ├── index.ts          Root reducer combining all slices
    ├── notification.ts   Show/hide notification state
    ├── sidebar.ts        Sidebar open/close state
    ├── todo.ts           Todo list with add/toggle/delete
    └── theme.ts          Theme preference state

lib/helper/storageLocal.ts  localStorage read/write helpers
```

## Setup

Wrap your app root with `StoreProvider`:

```tsx
// app/layout.tsx
import StoreProvider from "@/redux/StoreProvider"

export default function RootLayout({ children }: { children: React.ReactNode }) {
  return (
    <html>
      <body>
        <StoreProvider>{children}</StoreProvider>
      </body>
    </html>
  )
}
```

<Callout variant="info">
  `StoreProvider` is a `"use client"` component. It is safe to use inside the root layout — it only hydrates on the client side and does not break server rendering.
</Callout>

## Reading state

Use `useGetState` — a typed hook object where each selector is a method. Import it from `@/redux/connections`:

```tsx
"use client"

import { useGetState } from "@/redux/connections"

export function StatusBar() {
  const todos        = useGetState.getTodos()
  const count        = useGetState.getTodoCount()
  const completed    = useGetState.getCompletedCount()
  const notifOpen    = useGetState.getNotificationStatus()
  const notifMessage = useGetState.getNotificationMessage()
  const sidebarOpen  = useGetState.getSidebarStatus()

  return (
    <div>
      {count} todos ({completed} done) ·{" "}
      {notifOpen ? `Notif: ${notifMessage}` : "No notification"}
    </div>
  )
}
```

<Callout variant="tip">
  `useGetState` is fully typed — TypeScript will autocomplete all selector names and infer return types automatically.
</Callout>

## Dispatching actions

Use `dispatch` — a typed object where each action is a direct method call:

```tsx
"use client"

import { dispatch } from "@/redux/connections"

export function TodoActions() {
  return (
    <div className="flex gap-2">
      <button onClick={() => dispatch.addTodo("New task")}>
        Add Todo
      </button>
      <button onClick={() => dispatch.toggleTodo("todo-id-123")}>
        Toggle
      </button>
      <button onClick={() => dispatch.deleteTodo("todo-id-123")}>
        Delete
      </button>
      <button onClick={() => dispatch.clearTodos()}>
        Clear All
      </button>
    </div>
  )
}
```

### All available actions

| Action | Payload | Description |
|--------|---------|-------------|
| `dispatch.addTodo(text)` | `string` | Add a new todo item |
| `dispatch.toggleTodo(id)` | `string` | Toggle completed state |
| `dispatch.deleteTodo(id)` | `string` | Remove a todo by id |
| `dispatch.clearTodos()` | — | Reset the todo list |
| `dispatch.setNotificationShow(message)` | `string` | Show a notification |
| `dispatch.setNotificationHide()` | — | Hide the notification |
| `dispatch.updateNotification(message)` | `string` | Update notification text |
| `dispatch.setSidebarShow()` | — | Open the sidebar |
| `dispatch.setSidebarHide()` | — | Close the sidebar |

## Non-reactive state access

Use `getState` when you need the current value outside a React component (e.g. in an event handler or utility function):

```ts
import { getState } from "@/redux/connections"

// Read current state without subscribing to updates
const todos = getState.getTodos()
const isOpen = getState.getSidebarStatus()
```

## localStorage persistence

Selected slices are automatically saved to localStorage and restored on page load (SSR-safe — only runs in the browser). Saves are debounced at 120ms to avoid excessive writes.

Default persisted slices: `notifications`, `sidebar`, `todo`, `theme`.

To add a new slice to persistence, update `persistKeys` in `redux/store.ts`:

```ts
// redux/store.ts
export const persistKeys: StateKey[] = [
  "notifications",
  "sidebar",
  "todo",
  "theme",
  "myFeature", // ← add your slice key here
]
```

## Adding a new slice

Use `createReduxMethods` to define a slice — it returns a `[reducer, actions, selectors]` tuple with zero boilerplate:

```ts
// redux/reducers/myFeature.ts
import { createReduxMethods, ComposePayload } from "react-redux-methods"

type MyFeatureState = {
  value: string
  count: number
}

const initialState: MyFeatureState = { value: "", count: 0 }

const [myFeatureReducer, myFeatureActions, myFeatureSelectors] = createReduxMethods({
  initialState,
  reducers: {
    setValue: (state, action: ComposePayload<string>) => ({
      ...state,
      value: action.payload,
    }),
    increment: (state) => ({ ...state, count: state.count + 1 }),
    reset: () => initialState,
  },
  selectionNode: "myFeature",  // must match the key in rootReducer
  selectors: {
    getMyValue: (s) => s.value,
    getMyCount: (s) => s.count,
  },
})

export { myFeatureReducer, myFeatureActions, myFeatureSelectors }
```

Then register it in `redux/reducers/index.ts`:

```ts
import { myFeatureReducer, myFeatureActions, myFeatureSelectors } from "./myFeature"

export const rootReducer = combineReducers({
  // ...existing reducers
  myFeature: myFeatureReducer,
})

export const actions = {
  // ...existing actions
  ...myFeatureActions,
}

export const selectors = {
  // ...existing selectors
  ...myFeatureSelectors,
}

export const initialState = {
  // ...existing state
  myFeature: { value: "", count: 0 },
}
```

After registration, `dispatch.setValue("hello")` and `useGetState.getMyValue()` are immediately available — no extra wiring needed.

<Callout variant="tip">
  The `selectionNode` value must exactly match the key used in `combineReducers`. This is how `react-redux-methods` knows which slice of state to pass to each selector.
</Callout>
