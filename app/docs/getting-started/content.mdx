# Installation

Get your first FormKitCN block installed and running in under a minute.

## Prerequisites

Your project needs:

- Next.js 13+ (App Router)
- React 18+
- TypeScript
- Tailwind CSS
- shadcn/ui initialized — run `npx shadcn@latest init` if you haven't already

<Callout variant="info">
  FormKitCN components are shadcn registry blocks. The shadcn CLI handles all dependency installation automatically.
</Callout>

## Install your first block

Pick a block and add it to your project with one command. Here's the most common starting point — the dynamic form field:

<InstallCmd file="form-field.json" />

The CLI will copy the component files into your project and install all required npm packages.

<Callout variant="info">
  Install commands use the current registry domain. If you self-host FormKitCN, set `REGISTRY_BASE_URL` and `NEXT_PUBLIC_SITE_URL` so generated commands point at your deployed registry instead of localhost.
</Callout>

## Sonner setup

All FormKitCN blocks that use toast notifications include `sonner` as a registry dependency — it is installed automatically with the block. The only manual step is adding the `Toaster` to your root layout once:

```tsx
// app/layout.tsx
import { Toaster } from "@/components/ui/sonner"

export default function RootLayout({ children }: { children: React.ReactNode }) {
  return (
    <html lang="en">
      <body>
        {children}
        <Toaster position="top-center" />
      </body>
    </html>
  )
}
```

### Step 2 — Add the Toaster to your root layout

`sonner` was installed in step 1. Register it once in `app/layout.tsx`:

<Callout variant="warning">
  Make sure the font variables **and** `font-sans` are applied to `<body>` — not `<html>`. Without `font-sans`, Tailwind never reads the CSS variable and the browser falls back to a serif system font.
</Callout>

```tsx
import { Geist, Geist_Mono } from "next/font/google"
import { Toaster } from "@/components/ui/sonner"
import "./globals.css"

const geistSans = Geist({ variable: "--font-geist-sans", subsets: ["latin"] })
const geistMono = Geist_Mono({ variable: "--font-geist-mono", subsets: ["latin"] })

export default function RootLayout({ children }: { children: React.ReactNode }) {
  return (
    <html lang="en">
      <body className={`${geistSans.variable} ${geistMono.variable} font-sans antialiased`}>
        {children}
        <Toaster position="top-center" />
      </body>
    </html>
  )
}
```

### Step 3 — Use it in your component

<Callout variant="warning">
  Form components use React hooks (`useForm`, `useState`) — they must be Client Components. Add `"use client"` at the top of any file that renders a form.
</Callout>

```tsx
"use client"

import { useForm } from "react-hook-form"
import { zodResolver } from "@hookform/resolvers/zod"
import { z } from "zod"
import { Button } from "@/components/ui/button"
import { FieldRenderer  } from "@/components/forms/core"
import { FormFieldType } from "@/components/forms/core"

const schema = z.object({
  email: z.string().email(),
  password: z.string().min(8),
})

export function LoginForm() {
  const form = useForm({
    resolver: zodResolver(schema),
    defaultValues: { email: "", password: "" },
  })

  return (
    <form onSubmit={form.handleSubmit(console.log)} className="space-y-4">
      <FieldRenderer
        control={form.control}
        name="email"
        fieldType={FormFieldType.INPUT}
        label="Email"
        placeholder="you@example.com"
        required
      />
      <FieldRenderer
        control={form.control}
        name="password"
        fieldType={FormFieldType.PASSWORD}
        label="Password"
        required
      />
      <Button type="submit">Sign in</Button>
    </form>
  )
}
```

<Callout variant="tip">
  Want to skip the react-hook-form boilerplate? Use **SchemaForm** instead — it handles schema, layout, and submission automatically. See the [SchemaForm docs](/docs/form-builder).
</Callout>
