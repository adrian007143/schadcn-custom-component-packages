# MultiStepForm

A multi-step form component with per-step Zod validation, progress tracking, flexible layouts, and optional reCAPTCHA.

## Install

```bash
npx shadcn@latest add https://schadcn-custom-component-packages.vercel.app/r/multistep-form-template.json
```

## Usage

```tsx
import { MultiStepForm } from "@/components/forms/form-ui/multistep-form/MultiStepForm"
import { FormFieldType } from "@/components/forms/form-field/constants"
import { z } from "zod"

const step1Schema = z.object({
  orgName: z.string().min(1, "Required"),
  industry: z.string().min(1, "Required"),
})

const step2Schema = z.object({
  adminEmail: z.string().email(),
  adminName: z.string().min(1),
})

export function CreateOrgForm() {
  return (
    <MultiStepForm
      steps={[
        {
          title: "Organization",
          schema: step1Schema,
          fields: [
            { name: "orgName", fieldType: FormFieldType.INPUT, label: "Organization name" },
            { name: "industry", fieldType: FormFieldType.SELECT, label: "Industry", data: industries, labelKey: "label", valueKey: "value" },
          ],
        },
        {
          title: "Admin Account",
          schema: step2Schema,
          fields: [
            { name: "adminName", fieldType: FormFieldType.INPUT, label: "Admin name" },
            { name: "adminEmail", fieldType: FormFieldType.INPUT, label: "Admin email" },
          ],
        },
      ]}
      defaultValues={{ orgName: "", industry: "", adminEmail: "", adminName: "" }}
      onSubmit={async (values) => {
        await createOrg(values)
      }}
    />
  )
}
```

## Props

| Prop | Type | Required | Description |
|---|---|---|---|
| `steps` | `FormStep[]` | Yes | Array of step definitions |
| `defaultValues` | `DefaultValues<T>` | Yes | Combined initial values for all steps |
| `onSubmit` | `(values: T) => void \| Promise<void>` | Yes | Final submit handler |
| `layout` | `"auto" \| "horizontal" \| "vertical"` | — | Progress indicator layout |
| `recaptcha` | `boolean` | — | Enable reCAPTCHA on final step |
| `siteKey` | `string` | — | Google reCAPTCHA site key |

## Step Config

```ts
interface FormStep<T> {
  title: string
  description?: string
  schema: ZodType      // validated when "Next" is clicked
  fields: FormBuilderField<T>[]
  layout?: SectionLayout
}
```

## Per-Step Validation

Each step has its own `schema`. Clicking "Next" validates only the current step's fields against its schema. The user cannot advance until the current step is valid.

## reCAPTCHA

Set `recaptcha={true}` and `siteKey="YOUR_SITE_KEY"`. The reCAPTCHA token is verified before the final `onSubmit` is called.

```tsx
<MultiStepForm
  recaptcha
  siteKey={process.env.NEXT_PUBLIC_RECAPTCHA_SITE_KEY}
  ...
/>
```
