# StepForm

A multi-step form with per-step field validation, progress tracking, flexible layouts, and optional reCAPTCHA. Clicking "Next" validates only the current step's fields - users can't advance until the current step is valid.

## Install

<InstallCmd file="multistep-form-template.json" />

<Callout variant="tip">
  The registry install handles everything automatically - npm packages (`react-hook-form`, `@hookform/resolvers`, `zod`, `react-google-recaptcha`), shared registry blocks (`form-field`, `form-layout`), and shadcn components (`field`, `button`, `sonner`, `progress`). No manual installs needed.
</Callout>

After installing, add the `Toaster` once in your root layout. Make sure `font-sans` is on `<body>` or your app will render with a serif system font:

```tsx
// app/layout.tsx
import { Geist, Geist_Mono } from "next/font/google"
import { Toaster } from "@/components/ui/sonner"
import "./globals.css"

const geistSans = Geist({ variable: "--font-geist-sans", subsets: ["latin"] })
const geistMono = Geist_Mono({ variable: "--font-geist-mono", subsets: ["latin"] })

export default function RootLayout({ children }: { children: React.ReactNode }) {
  return (
    <html lang="en">
      <body className={`${geistSans.variable} ${geistMono.variable} font-sans antialiased`}>
        {children}
        <Toaster position="top-center" />
      </body>
    </html>
  )
}
```

## Usage

<Callout variant="warning">
  `StepForm` uses React hooks internally - add `"use client"` to any file that renders it.
</Callout>

```tsx
"use client"

import { StepForm } from "@/components/forms/form-ui/multistep-form"
import { FormFieldType } from "@/components/forms/core"
import { z } from "zod"

const schema = z.object({
  orgName: z.string().min(1, "Required"),
  industry: z.string().min(1, "Required"),
  adminEmail: z.string().email(),
  adminName: z.string().min(1, "Required"),
})

type FormValues = z.infer<typeof schema>

export function CreateOrgForm() {
  return (
    <StepForm<FormValues>
      schema={schema}
      defaultValues={{ orgName: "", industry: "", adminEmail: "", adminName: "" }}
      steps={[
        {
          title: "Organization",
          description: "Tell us about your organization",
          fields: [
            { name: "orgName", fieldType: FormFieldType.INPUT, label: "Organization name" },
            { name: "industry", fieldType: FormFieldType.SELECT, label: "Industry",
              data: industries, labelKey: "label", valueKey: "value" },
          ],
        },
        {
          title: "Admin Account",
          description: "Who will manage this organization?",
          layout: "grid-2",
          fields: [
            { name: "adminName", fieldType: FormFieldType.INPUT, label: "Full name" },
            { name: "adminEmail", fieldType: FormFieldType.INPUT, label: "Email address" },
          ],
        },
      ]}
      onSubmit={async (values, form) => {
        await createOrg(values)
      }}
    />
  )
}
```

## Step config

```ts
interface StepFormStep<T> {
  id?: string                    // Optional key (defaults to index)
  title: string                  // Shown in the step indicator
  description?: string           // Optional sub-title for the step
  fields?: StepFieldConfig<T>[]  // Flat list of fields for this step
  sections?: FormSection<T>[]    // Use sections for multi-column layouts within a step
  layout?: FormSectionLayout     // Grid layout when using fields (not sections)
  columns?: number               // Column count when layout is "grid"
}
```

<Callout variant="info">
  Use `fields` for a simple flat list. Use `sections` when you need multiple layout groups within a single step. You can't use both at the same time.
</Callout>

## Per-step validation

The full schema is provided once at the top level. When "Next" is clicked, `StepForm` triggers validation only for the field names present in the current step - users cannot advance until those fields are valid.

```ts
// Define the complete schema upfront
const schema = z.object({
  // step 1 fields
  orgName: z.string().min(1, "Required"),
  industry: z.string().min(1, "Required"),
  // step 2 fields
  adminEmail: z.string().email(),
  adminName: z.string().min(1, "Required"),
})
```

Each step's `fields[].name` values must match keys in that schema object.

## Layout modes

```tsx
// Auto - sidebar on desktop, top bar on mobile (default)
<StepForm layoutMode="auto" ... />

// Always sidebar (steps shown left of the form)
<StepForm layoutMode="horizontal" ... />

// Always top bar (steps shown above the form)
<StepForm layoutMode="vertical" ... />
```

## Sections within a step

Use `sections` instead of `fields` when you need multiple layout groups in one step:

```tsx
{
  title: "Company Details",
  sections: [
    {
      title: "Basic info",
      layout: "grid-2",
      fields: [
        { name: "companyName", fieldType: FormFieldType.INPUT, label: "Name" },
        { name: "companySize", fieldType: FormFieldType.SELECT, label: "Size",
          data: sizes, labelKey: "label", valueKey: "value" },
      ],
    },
    {
      title: "Address",
      layout: "grid-3",
      fields: [
        { name: "city", fieldType: FormFieldType.INPUT, label: "City" },
        { name: "state", fieldType: FormFieldType.INPUT, label: "State" },
        { name: "zip", fieldType: FormFieldType.INPUT, label: "ZIP" },
      ],
    },
  ],
}
```

## reCAPTCHA

Set `recaptcha={true}` - the widget appears on the final step above the Submit button. Submission is blocked until the user verifies.

```tsx
<StepForm
  recaptcha
  schema={schema}
  defaultValues={defaults}
  steps={steps}
  onSubmit={handleSubmit}
/>
```

Add your site key to `.env.local`:

```bash
NEXT_PUBLIC_RECAPTCHA_SITE_KEY=your_site_key_here
```

## Cancel button

Pass `onCancel` to show a Cancel button on every step:

```tsx
<StepForm
  onCancel={() => router.back()}
  ...
/>
```
