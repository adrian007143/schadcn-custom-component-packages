# Theme Preset Tool

An interactive theme panel with 12 OKLCH presets, per-token color editor, radius control, and CSS/JSON export. Standalone — installs its own lightweight store with no Redux required in consumer projects.

## Install

<InstallCmd file="theme-preset-tool.json" />

<Callout variant="tip">
  The install automatically includes `lucide-react`, all required shadcn/ui components, and the `redux-methods-tool` dependency.
</Callout>

```bash
styles/
└── theme-preset-base.css         # base html/body layout styles (see Step 1)

components/
├── providers/
│   └── AppProviders.tsx          # wraps theme provider + panel + toaster
├── site/
│   ├── AppHeader.tsx             # pre-wired header (trigger + toggle)

│   └── ThemeToggle.tsx           # light / dark toggle button
└── theme/
    ├── app-theme-provider.tsx    # custom theme context (replaces next-themes)
    ├── ThemeManager.tsx          # required — syncs store → CSS variables
    ├── ThemeBuilder.tsx          # slide-out preset panel

├── types.ts
└── utils.ts
```

## Setup — shadcn nova preset (Next.js 16)

This guide covers the setup when you initialised your project with:

```bash
npx shadcn@latest init
# → "Yes, use recommended defaults" → "nova" preset
```

Follow the two steps below in order.

---

### Step 1 — Fix globals.css

The nova preset generates `--font-sans: var(--font-sans)` in `@theme inline` — a self-referencing variable that breaks the font stack. Fix it by pointing to the actual Geist CSS variable:

```css
/* app/globals.css */
@import "tailwindcss";
@import "tw-animate-css";
@import "shadcn/tailwind.css";

@custom-variant dark (&:is(.dark *));

@theme inline {
  --color-background: var(--background);
  --color-foreground: var(--foreground);

  /* Fix: was var(--font-sans) — self-referencing, broken */
  --font-sans: var(--font-geist-sans);
  --font-mono: var(--font-geist-mono);

  /* ... rest of your @theme inline tokens unchanged ... */
}

/* :root and .dark blocks stay exactly as generated by nova */
```

<Callout variant="tip">
  The `@custom-variant dark (&:is(.dark *))` line is already in the nova preset. The theme provider adds `.dark` to `<html>` on the client, which triggers this variant for all descendant elements — no extra configuration needed.
</Callout>

---

### Step 2 — Root layout

<Callout variant="warning">
  **`suppressHydrationWarning` is required on `<html>`.** The theme provider adds `.dark` or `.light` to `<html>` on the client. Without this attribute React throws a hydration warning on every page load.
</Callout>

<Callout variant="warning">
  **All providers must go inside `<body>`.** Placing a `<div>` directly inside `<html>` (without `<body>`) is invalid HTML and causes: `In HTML, <div> cannot be a child of <html>`.
</Callout>

```tsx
// app/layout.tsx
import type { Metadata } from "next"
import { Geist, Geist_Mono } from "next/font/google"
import { AppProviders } from "@/components/providers/AppProviders"
import { AppHeader } from "@/components/site/AppHeader"
import { AppShell } from "@/components/site/AppShell"
import "./globals.css"

const geistSans = Geist({
  variable: "--font-geist-sans",
  subsets: ["latin"],
})

const geistMono = Geist_Mono({
  variable: "--font-geist-mono",
  subsets: ["latin"],
})

export const metadata: Metadata = {
  title: "My App",
  description: "Application shell for My App.",
}

export default function RootLayout({
  children,
}: Readonly<{
  children: React.ReactNode
}>) {
  return (
    <html
      lang="en"
      suppressHydrationWarning
      className={`${geistSans.variable} ${geistMono.variable} h-full`}
    >
      <body className="flex min-h-dvh flex-col bg-background font-sans text-foreground antialiased">
        <AppProviders defaultTheme="dark">
          <AppShell header={<AppHeader brand="My App" />}>
            {children}
          </AppShell>
        </AppProviders>
      </body>
    </html>
  )
}
```

**Why these specific classes:**
- `h-full` on `<html>` — ensures the root element participates in full-height layout
- `min-h-dvh` on `<body>` — uses dynamic viewport height (correct on mobile, avoids address-bar clipping)
- `flex flex-col` on `<body>` — makes `AppShell`'s `flex-1` stretch to fill the full viewport
- `bg-background font-sans text-foreground antialiased` — base visual styling

**With Redux** — wrap `AppProviders` with `StoreProvider` inside `<body>`:

```tsx
import StoreProvider from "@/redux/StoreProvider"

export default function RootLayout({
  children,
}: Readonly<{
  children: React.ReactNode
}>) {
  return (
    <html
      lang="en"
      suppressHydrationWarning
      className={`${geistSans.variable} ${geistMono.variable} h-full`}
    >
      <body className="flex min-h-dvh flex-col bg-background font-sans text-foreground antialiased">
        <StoreProvider>
          <AppProviders defaultTheme="dark">
            <AppShell header={<AppHeader brand="My App" />}>
              {children}
            </AppShell>
          </AppProviders>
        </StoreProvider>
      </body>
    </html>
  )
}
```

## Other setup types

### Blank Next.js app (no shadcn preset)

```tsx
// app/layout.tsx
import { AppProviders } from "@/components/providers/AppProviders"
import { AppHeader } from "@/components/site/AppHeader"
import { AppShell } from "@/components/site/AppShell"
import "./globals.css"

export default function RootLayout({ children }: { children: React.ReactNode }) {
  return (
    <html lang="en" suppressHydrationWarning>
      <body className="flex min-h-dvh flex-col bg-background text-foreground antialiased">
        <AppProviders defaultTheme="dark">
          <AppShell header={<AppHeader brand="My App" />}>
            {children}
          </AppShell>
        </AppProviders>
      </body>
    </html>
  )
}
```

Add the base CSS to `globals.css` (handles `--font-sans` and any missing Tailwind tokens):

```css
@import "tailwindcss";
@import "./styles/theme-preset-base.css";
```

### Minimal (ThemeManager only, no panel)

Use this if you only want preset token support without the slide-out panel UI.

```tsx
import { AppThemeProvider } from "@/components/theme/app-theme-provider"
import { ThemeManager } from "@/components/theme/ThemeManager"

export default function RootLayout({ children }: { children: React.ReactNode }) {
  return (
    <html lang="en" suppressHydrationWarning>
      <body className="flex min-h-dvh flex-col bg-background text-foreground antialiased">
        <AppThemeProvider attribute="class" defaultTheme="dark">
          <ThemeManager />
          {children}
        </AppThemeProvider>
      </body>
    </html>
  )
}
```

<Callout variant="warning">
  `ThemeManager` must always be present inside `AppThemeProvider`. Without it, preset changes will never apply — it is the component that writes CSS custom properties to `document.documentElement` on every state update.
</Callout>

## dispatchThemeAction

```tsx
import { dispatchThemeAction } from "@/components/theme/dispatchThemeAction"
```

### Open / close the panel

```tsx
dispatchThemeAction("setThemeBuilderOpen", true)
dispatchThemeAction("setThemeBuilderOpen", false)
```

### Switch preset

```tsx
dispatchThemeAction("setThemePreset", "midnight")
dispatchThemeAction("setThemePreset", "ocean")
dispatchThemeAction("setThemePreset", "default")  // uses globals.css tokens
```

### Override individual tokens

```tsx
dispatchThemeAction("setCustomLight", { key: "--primary", value: "oklch(0.52 0.16 240)" })
dispatchThemeAction("setCustomDark",  { key: "--primary", value: "oklch(0.65 0.18 240)" })
dispatchThemeAction("clearCustomLight", "--primary")
dispatchThemeAction("clearCustomDark",  "--primary")
```

### Radius, reset, import

```tsx
dispatchThemeAction("setRadius", 0.5)
dispatchThemeAction("resetTheme")
dispatchThemeAction("importThemeOverrides", {
  preset: "ocean",
  customLight: {},
  customDark: {},
  radius: 0.75,
})
```

## Themed Scrollbar (Optional)

Add themed scrollbars to your app so the page scroll and sidebar scroll both match the active light/dark preset. This is done entirely in `globals.css` — no JavaScript required.

### Step 1 — Add scrollbar color tokens

Add `--scrollbar-thumb` and `--scrollbar-thumb-hover` to your `:root` and `.dark` blocks:

```css
/* app/globals.css */

:root {
  /* ... your existing tokens ... */
  --scrollbar-thumb:       oklch(0.78 0.01 255);   /* light mode — medium gray */
  --scrollbar-thumb-hover: oklch(0.58 0.02 255);
}

.dark {
  /* ... your existing tokens ... */
  --scrollbar-thumb:       oklch(0.38 0.03 265);   /* dark mode — subtle blue-gray */
  --scrollbar-thumb-hover: oklch(0.55 0.04 265);
}
```

<Callout variant="tip">
  Adjust the lightness value (first number) to taste. On a dark sidebar (`oklch(0.208 ...)`) a thumb around `0.35–0.42` is visible but not distracting. On a light background (`oklch(0.984 ...)`) aim for `0.72–0.82`.
</Callout>

### Step 2 — Add scrollbar rules inside `@layer base`

```css
/* app/globals.css */

@layer base {
  /* ... your existing base rules ... */

  /* Page scroll + sidebar scroll — themed to match light/dark mode */
  html,
  [data-slot="sidebar-content"],
  [data-sidebar="content"],
  .scrollbar-themed {
    scrollbar-width: thin;
    scrollbar-color: var(--scrollbar-thumb) transparent;
  }

  html::-webkit-scrollbar,
  [data-slot="sidebar-content"]::-webkit-scrollbar,
  [data-sidebar="content"]::-webkit-scrollbar,
  .scrollbar-themed::-webkit-scrollbar {
    width: 5px;
    height: 5px;
  }

  html::-webkit-scrollbar-track,
  [data-slot="sidebar-content"]::-webkit-scrollbar-track,
  [data-sidebar="content"]::-webkit-scrollbar-track,
  .scrollbar-themed::-webkit-scrollbar-track {
    background: transparent;
  }

  html::-webkit-scrollbar-thumb,
  [data-slot="sidebar-content"]::-webkit-scrollbar-thumb,
  [data-sidebar="content"]::-webkit-scrollbar-thumb,
  .scrollbar-themed::-webkit-scrollbar-thumb {
    background-color: var(--scrollbar-thumb);
    border-radius: 9999px;
  }

  html::-webkit-scrollbar-thumb:hover,
  [data-slot="sidebar-content"]::-webkit-scrollbar-thumb:hover,
  [data-sidebar="content"]::-webkit-scrollbar-thumb:hover,
  .scrollbar-themed::-webkit-scrollbar-thumb:hover {
    background-color: var(--scrollbar-thumb-hover);
  }
}
```

**What each selector covers:**

<PropsTable items={[
  { name: "html", type: "selector", description: "The outer page scroll — the scrollbar visible on the right edge of the viewport" },
  { name: "[data-slot=\"sidebar-content\"]", type: "selector", description: "shadcn/ui Sidebar scroll area (primary attribute)" },
  { name: "[data-sidebar=\"content\"]", type: "selector", description: "shadcn/ui Sidebar scroll area (secondary attribute on the same element)" },
  { name: ".scrollbar-themed", type: "utility class", description: "Apply to any other scrollable container — modals, panels, drawers, etc." },
]} />

### Using `.scrollbar-themed` on custom containers

Add the class to any `overflow-auto` / `overflow-y-auto` element:

```tsx
<div className="scrollbar-themed h-96 overflow-y-auto">
  {/* long content */}
</div>
```

## Troubleshooting

### `<div> cannot be a child of <html>`

All providers render `<div>` elements. They must go inside `<body>`, not directly inside `<html>`.

```tsx
// ❌ Wrong
<html lang="en">
  <AppProviders>...</AppProviders>   {/* <div> directly inside <html> */}
</html>

// ✅ Correct
<html lang="en" suppressHydrationWarning>
  <body>
    <AppProviders>...</AppProviders>
  </body>
</html>
```

### Theme toggle causes hydration warning even with `suppressHydrationWarning`

Make sure `suppressHydrationWarning` is on `<html>`, not `<body>`. The theme provider modifies the `<html>` class attribute.

```tsx
<html lang="en" suppressHydrationWarning>   {/* ← must be here */}
  <body>
```

### Dark mode classes not applying (nova preset)

Verify `globals.css` contains:

```css
@custom-variant dark (&:is(.dark *));
```

This must appear before any `@theme inline` or `:root` blocks.

### Theme changes do not apply after install

1. `ThemeManager` must be mounted inside `AppThemeProvider` in the root layout.
2. Your page shell must use `bg-background text-foreground` (not hardcoded colors).
3. Delete `.next` and restart `npm run dev`.

### Panel opens but content sits behind it

Use `AppShell`, or read `isBuilderOpen` and add a right offset manually:

```tsx
import { useThemeBuilderState } from "@/components/theme/useThemeBuilderState"

const { isBuilderOpen } = useThemeBuilderState()

<main className={isBuilderOpen ? "lg:pr-[446px]" : ""}>
  {children}
</main>
```

### Font not applying with nova preset

The nova preset generates `--font-sans: var(--font-sans)` in `@theme inline` — a self-referencing variable that breaks the font stack. Fix it in `globals.css` by pointing to the real variable:

```css
@theme inline {
  --font-sans: var(--font-geist-sans);  /* ← was: var(--font-sans) — broken */
  --font-mono: var(--font-geist-mono);
}
```

Make sure your font loader uses the matching variable name:

```tsx
const geistSans = Geist({
  variable: "--font-geist-sans",   // ← must match what @theme inline references
  subsets: ["latin"],
})
```
